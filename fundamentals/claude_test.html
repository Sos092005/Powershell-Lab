Ce guide est conçu pour être suivi phase par phase, dans l'ordre. Chaque phase produit un résultat concret et opérationnel. Aucune phase ne dépend d'un outil payant — l'intégralité du système repose sur les plans gratuits de GitHub.

Phase 1

L'Identité

Cette phase crée le "cerveau" de votre projet : l'Organisation GitHub, avec une structure de fichiers optimisée pour la recherche par matière. À la fin de cette phase, vous disposerez d'un espace collectif sécurisé, prêt à accueillir les archives de la filière.

Étape 0 — Pré-requis

Avant de commencer, assurez-vous de disposer des éléments suivants :

  • Un compte GitHub personnel (plan gratuit suffisant).
  • Le nom définitif du club, par exemple : Club-Info-Univ-X.
💡 Tip

Choisissez un nom d'organisation court, sans accents et sans espaces. Ce nom apparaîtra dans toutes vos URLs GitHub et sera visible par tous les membres.

Étape 1 — Création de l'Espace Collectif

  1. Connexion — Rendez-vous sur github.com et connectez-vous à votre compte personnel.
  2. Nouvelle Organisation — Cliquez sur le signe + en haut à droite, puis sélectionnez "New organization".
  3. Choix du Plan — Sélectionnez le plan "Free".
  4. Configuration — Renseignez les champs suivants :
Configuration de l'organisation GitHub
ChampValeur attendue
Organization account nameInfo-Filiere-X
Contact emailEmail du club ou du délégué
Belongs toMy personal account
  1. Validation — Cliquez sur "Next", puis "Finish setup".

Étape 2 — Création du Dépôt "Coffre-Fort"

  1. Depuis la page d'accueil de votre organisation, cliquez sur l'onglet "Repositories", puis sur le bouton vert "New repository".
  2. Renseignez les paramètres du dépôt :
Paramètres du dépôt principal
ParamètreValeur
Repository namearchives-sujets

Initialize✅ Add a README file
⚠️ Warning

Ne passez jamais ce dépôt en Public. Les sujets d'examens sont des ressources sensibles, réservées aux membres de la filière.

  1. Cliquez sur le bouton vert "Create repository".

Étape 3 — Arborescence Intuitive (Le Rangement)

La logique de rangement adoptée est : Matière → Session → Année. Voici comment créer la structure vide pour l'Algorithmique :

  1. Cliquez sur "Add file" > "Create new file".
  2. Dans le champ nom du fichier, tapez le chemin suivant (les / créent automatiquement les dossiers) :
Chemin à saisir dans GitHub
Matieres/Algorithmique/Session-Normale/2025-2026/.gitkeep
📝 Note

Le fichier .gitkeep est un fichier vide dont le seul rôle est de permettre à GitHub de sauvegarder un dossier vide. Il sera remplacé dès le premier vrai dépôt de PDF.

  1. Cliquez sur "Commit changes".

Résumé de phase

Vous disposez désormais d'un espace collectif GitHub, d'un dépôt privé sécurisé et d'une arborescence prête à recevoir les fichiers.

Phase suivante : La Gouvernance — configuration des équipes, permissions et labels visuels.
Phase 2

La Gouvernance

L'objectif de cette phase est de transformer un projet "solo" en un projet collectif sécurisé, avec des rôles clairement définis. Sur GitHub, les accès ne se gèrent pas individu par individu : on crée des Teams auxquelles on attribue des permissions globales.

Étape 1 — Création de l'Équipe de Direction (The Team)

  1. Allez sur la page d'accueil de votre Organisation (cliquez sur le logo du club).
  2. Cliquez sur l'onglet "Teams".
  3. Cliquez sur le bouton vert "New team" et renseignez les champs suivants :
Configuration de l'équipe de direction
ChampValeur
Team nameBureau-du-Club
DescriptionMembres responsables de la validation des sujets
VisibilityVisible
  1. Cliquez sur "Create team".

Étape 2 — Attribution des Permissions au "Coffre-Fort"

  1. Dans votre équipe Bureau-du-Club, cliquez sur l'onglet "Repositories".
  2. Cliquez sur "Add repository" et sélectionnez archives-sujets.
  3. Définissez le niveau de permission sur "Maintain".
ℹ️ Pourquoi "Maintain" et pas "Admin" ?

Le niveau Maintain permet aux membres du Bureau d'ajouter, organiser et gérer le contenu du dépôt. Il leur interdit cependant de supprimer le dépôt ou de modifier les réglages critiques. Seul le compte Owner (le coordinateur) conserve ces droits.

  1. Cliquez sur "Save".

Étape 3 — Invitation des Membres

  1. Dans l'onglet "Members" de l'équipe, cliquez sur "Add a member".
  2. Saisissez le nom d'utilisateur GitHub ou l'adresse email de chaque collègue.
  3. Envoyez l'invitation — le membre recevra un email de confirmation.

Étape 4 — Configuration des Labels (La Perception Visuelle)

Les Labels permettent d'identifier l'état d'un sujet sans ouvrir son dossier. Ils s'appliquent aux Issues (demandes de dépôt).

  1. Allez dans le dépôt archives-sujets > onglet "Issues" > bouton "Labels".
  2. Créez les trois labels standards suivants :

OK Labels configurés (signalétique visuelle opérationnelle)

Résumé de phase

Le projet dispose désormais d'une structure humaine avec des rôles clairs et une signalétique visuelle pour le suivi des contributions.

Phase suivante : Le Protocole de Dépôt — formulaire de contribution et mise à jour du README.
Phase 3

Le Protocole de Dépôt

Cette phase installe le "formulaire de réception" qui garantit que toutes les contributions arrivent dans un format uniforme dès le premier jour. Un étudiant qui ne sait pas comment contribuer ne contribuera pas — ce formulaire élimine ce frein.

📝 Note

Sur GitHub, les Issues ne servent pas uniquement à signaler des bugs. Dans notre contexte, elles constituent le canal officiel de dépôt de sujets par les étudiants.

Étape 1 — Activer les Modèles de Demandes (Issue Templates)

  1. Allez dans votre dépôt archives-sujets > onglet "Settings".
  2. Dans la section "Features", repérez "Issues" et cliquez sur "Set up templates".
  3. Cliquez sur "Add template" > "Custom template".
  4. Cliquez sur "Preview and edit".

Étape 2 — Configurer le Formulaire de Dépôt

Métadonnées du template
ChampValeur
Template name📦 Dépôt de Nouveau Sujet
AboutUtilisez ce formulaire pour nous envoyer un scan d'examen.

Remplacez le contenu du champ Template content par ce texte :

Template content — Markdown
### 📚 Informations sur le Sujet
- **Matière :** [Ex: Algorithmique]
- **Année :** [Ex: 2024-2025]
- **Session :** [Normale / Rattrapage]
- **Niveau :** [L1 / L2 / L3 / M1 / M2]

### 📎 Fichier
- [ ] J'ai glissé-déposé le PDF ou l'image ci-dessous.
- [ ] Le scan est lisible et complet.

### 📝 Remarques (Optionnel)
(Ex: Manque la question 3, ou sujet difficile cette année.)

Étape 3 — Automatisation des Labels

  1. Dans l'éditeur de template (colonne de droite), cliquez sur "Labels".
  2. Sélectionnez Besoin-de-Correction, ou créez Nouveau-Dépôt.
  3. Cliquez sur "Propose changes", puis "Commit changes".

Phase suivante : Validation et Archivage — du dépôt étudiant au dossier définitif.
Phase 4

Validation et Archivage

L'objectif de cette phase est de traiter les dépôts des étudiants pour les intégrer officiellement dans l'arborescence de la filière. C'est le cœur du workflow quotidien du Bureau du Club.

Étape 1 — Réception et Tri (Le rôle du Modérateur)

  1. Allez dans l'onglet "Issues" de votre dépôt.
  2. Cliquez sur la nouvelle demande reçue.
  3. Contrôle qualité — Vérifiez : lisibilité du scan, pages manquantes, métadonnées correctes.
  4. Si le fichier est conforme, ajoutez le commentaire :
💬 Commentaire de validation type

"Merci ! Sujet validé. En cours d'archivage par le Bureau."

Étape 2 — L'Archivage Physique (Le Transfert)

  1. Allez dans l'onglet "Code" du dépôt.
  2. Naviguez jusqu'à la destination finale :
Chemin de destination
Matieres/Algorithmique/Session-Normale/2025-2026/
  1. Cliquez sur "Add file" > "Upload files".
  2. Glissez-déposez le PDF transmis par l'étudiant.
  3. Nomenclature stricte — Vérifiez le nom : Algo-SN-2025.pdf
  4. Cliquez sur "Commit changes".
⚠️ Warning

Ne jamais renommer un fichier après archivage si un lien a déjà été partagé dans le README. Tout renommage brise les liens existants pour tous les étudiants.

Étape 3 — Fermeture de la Demande

  1. Retournez dans "Issues" et ouvrez la demande correspondante.
  2. Mettez à jour le Label : Correction-Disponible si la correction est jointe, sinon laisser le label actuel.
  3. Cliquez sur "Close with comment" et écrivez :
💬 Commentaire de clôture type

"Fichier ajouté avec succès à l'archive ! Merci pour ta contribution. 🎉"
